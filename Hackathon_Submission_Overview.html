2. Business Problem Statement

What problem exists?
Many users find it cumbersome to log into dashboards or switch apps to do simple banking (check balance, send money, see recent transactions). In messaging-first regions, people already use WhatsApp/Telegram; forcing them to leave the chat to bank creates friction and lowers engagement.

Why is it important?
Friction leads to abandoned transfers, delayed payments, and poor experience. Businesses and individuals need fast, low-friction access to Paystack operations from the channels they already use.

How does it affect customers or the business?
Customers face extra steps and context-switching; businesses see lower completion rates and support load from "how do I…" questions. A conversational interface in WhatsApp/Telegram reduces steps and meets users where they are.

3. Proposed Solution

What automation did you build?

  • Event-driven banking: Incoming WhatsApp/Telegram message triggers the flow. The system parses intent (balance, transfer, history, beneficiaries), runs the right Paystack operations, and replies in chat. Multi-step flows (e.g. transfer + OTP confirmation) are handled via conversation state and memory (MongoDB).
  • CLI automation: Interactive menu (Rich/InquirerPy) for the same operations so power users or support can run them from the terminal without the dashboard.

What tools were integrated?

  • Paystack API (balance, banks, resolve account, transfer recipients, initiate/finalize transfer, transactions)
  • Twilio (WhatsApp), Telegram Bot API (Telegram)
  • OpenRouter (LLM for intent and natural language), LangChain/LangGraph (orchestration)
  • MongoDB (conversation state, memory, recipient patterns)
  • FastAPI (webhooks, REST, receipt serving), Python (CLI and backend)

Why this approach (instead of n8n)?
We chose a custom API + AI agent stack to keep full control over conversation state, multi-step transfer flows, and Paystack integration, and to support both chat (WhatsApp/Telegram) and CLI from one codebase. The automation is event-driven (webhook as trigger) with clear processing and output stages.

What makes your solution effective?

  • Single place to bank from WhatsApp or Telegram (and CLI).
  • Natural language (e.g. "send 5k to mom", "what's my balance?") in English and Pidgin.
  • Memory and patterns (e.g. frequent recipients) for faster transfers and spending insights.
  • Optional receipts and OCR.
  • Security: webhook signature validation (Twilio), optional API key/JWT for REST, confirmation for sensitive actions.

4. Workflow Architecture (Trigger to Output)

Workflow implemented in-app (no n8n); structure is equivalent to Trigger → Processing → Output.

Trigger

  • Chat: Twilio or Telegram sends POST to webhook (/whatsapp/webhook or /telegram/webhook) with the user message.
  • CLI: User runs python main.py --cli and selects an action from the menu.

Processing

  • Chat path: Webhook → WhatsAppService / TelegramService → FinancialAgent.process_message() → MessageProcessor (intent/entities) → BalanceHandler, TransferHandler, HistoryHandler, BeneficiaryHandler, or AIHandler → PaystackService (and optionally recipient_manager, memory_manager). Multi-step state and OTP in ConversationState and MongoDB.
  • CLI path: Menu choice → same PaystackService methods with prompts (e.g. bank code, account number, amount).

Output

  • Chat: Formatted reply (and optional receipt image) sent back via Twilio/Telegram.
  • CLI: Rich tables/panels in the terminal and, for transfers, success/failure message.

5. Narrative

TizLion AI Banking is a fintech solution that automates Paystack operations over WhatsApp and Telegram and via a CLI. It addresses the friction of switching to a browser or app to check balance, send money, or view history. When a user sends a message (trigger), the system parses intent, runs the right Paystack API calls and stateful flows (processing), and replies in chat or in the terminal (output). Tools used include Paystack, Twilio, Telegram, OpenRouter, LangChain, MongoDB, and FastAPI. The solution is effective because it meets users in the channels they already use, supports natural language in English and Pidgin, and keeps state for multi-step transfers and OTP, with optional receipts and OCR.
